
CampusSync Product Requirements Document (PRD)

1. Product Overview
CampusSync is a web-based academic scheduling and coordination system designed for university environments.
The system centralises academic events, supervision bookings, and schedule communication between students and lecturers.
The primary goal is to reduce fragmentation of academic scheduling information and provide a unified platform for academic event management.
The system is implemented as a web application hosted on nuwebspace.

5. Functional Requirements

Must Have

FR1
Unified academic calendar displaying relevant academic events.

FR2
Lecturers can create, update, and manage academic events.

FR3
Students receive notifications when events are added or modified.

FR4
Events displayed based on user role and access permissions.

FR5
Students can view events related only to their enrolled modules.


Should Have

FR6
Centralised communication of schedule changes.

FR7
Quick access to frequently required academic information.

FR10
Filtering of events by module or event type.

FR11
Search functionality for locating specific academic events.


Could Have

FR8
Lecturers publish supervision slots.

FR9
Students book supervision slots.

FR12
Maintain record of previously created academic events.


Won't Have

FR13
Administrator role management.

FR14
External calendar integration.

8. Success Criteria

The system will be considered successful if users can:

1. Log in to the platform.
2. View academic events in a unified calendar.
3. Lecturers can create and manage events.
4. Students receive notifications for event changes.
5. Students can book available supervision slots.
6. Events can be searched and filtered.
