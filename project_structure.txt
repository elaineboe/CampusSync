
CampusSync System Project Structure

This document defines the required folder structure, technology stack, and deployment constraints for the CampusSync system.
The AI must follow this structure when generating code.
The system must be compatible with the Nuwebspace hosting environment.

--------------------------------------------------

1. Technology Stack

Frontend
React + Vite

Backend
PHP REST API

Database
MySQL

Deployment Environment
Nuwebspace hosting

File Transfer
WinSCP

Database Management
phpMyAdmin

--------------------------------------------------

2. Deployment URL

http://w25037992.nuwebspace.co.uk

--------------------------------------------------

3. System Architecture

The system follows a three-layer architecture.

Presentation Layer
React frontend responsible for UI rendering.

Application Layer
PHP backend API responsible for business logic.

Data Layer
MySQL database responsible for persistent data storage.

System communication flow:

Browser
→ React frontend
→ API request
→ PHP backend
→ MySQL database
→ response returned to frontend

5. Backend Configuration

Backend must be implemented using PHP REST API.
Node.js or Express servers must NOT be generated because Nuwebspace does not support Node servers.

Database configuration example:

return [
 'db_host' => 'nuwebspace_db',
 'db_name' => 'w25037992',
 'db_user' => 'w25037992',
 'db_pass' => '********'
];

10. Deployment Rules

Frontend must be built using:
npm run build

The built files must be uploaded to Nuwebspace using WinSCP.

Backend PHP files must be placed in the server directory.

Database schema must be imported using phpMyAdmin.
